Sparkles

Argument Defaults

Problem

The glimmer VM doesn't provide @argument defaults - that is by design.

Solution

Use the @arg decorator from sparkles-decorators.

Example

See Element Reference below.

Element Reference

Problem

In glimmer, components can access this.bounds to access the first and last element from the component at root level (which can be the same one). Sparkles as proof-of-concept implementation do not provide that access.

Solution

The official solution to this is being discussed in RFC #415: Render Element Modifiers.

Workaround

Until this is finalized and implemented, we need to rely on a workaround to access elements from the component. 
import SparklesComponent from 'sparkles-component';
import { guidFor } from '@ember/object/internals';
import { arg } from 'sparkles-decorators';

interface Args {
  id?: string
}

export default class ElementReferenceComponent extends SparklesComponent<Args> {
  @arg id: string = guidFor(this);

  didInsertElement() {
    document.getElementById(this.id);
  }
}

<div id={{this.id}}>
  {{yield}}
</div>

Example

No attributes with component helper

Problem

With angle-brackets, there are now attributes and arguments that can be passed in to a component. The component helper uses the attribute syntax for arguments and is itself incapable of providing support for this.

Solution

A solution is on the horizon that will pretty much look like the workaround we have to use for now.

Workaround

Use ember-component-attributes addon. It will provide a html-attributes helper for usage inside component helper.
